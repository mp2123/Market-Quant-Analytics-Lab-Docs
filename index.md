## Recruiter Brief

This hub organizes one lane of Michael's public proof: Python market-data ingestion, SQLite-backed local data organization, dashboard review, source-governance discipline, expected-move and volatility research, and quant-adjacent validation work.

The point is not to claim a live trading platform. The point is to show practical financial-data curiosity, analytics engineering habits, dashboard judgment, and careful separation between public documentation and private runtime state.

## What This Hub Is

This is a navigation and boundary layer. It explains how the public proof shells fit together while keeping implementation repos, credentials, local databases, production runtime state, and account-specific records private.

## Public Ecosystem

| Layer | Public role |
| --- | --- |
| Institutional Market Data Engine | Python ingestion, SQLite storage, dashboard review, source-governance discipline. |
| Market Data Dashboard | Streamlit-style dashboarding, observability, validation boundaries, private runtime separation. |
| QuantStrat ML | Backtesting, expected-move research, volatility analytics, no-lookahead validation. |
| Portfolio Website | Recruiter-facing narrative, sanitized proof pages, resume and project routing. |

## Private Boundary

Private production/runtime material remains private. This hub does not expose API credentials, OAuth tokens, SQLite databases, broker state, private watchlists, local paths, monitor payloads, Discord/n8n/TradingView state, or account-specific artifacts.

## What It Does Not Claim

This is not a live trading system, broker automation platform, professional quant platform, or investment-advice product. It is a public-safe proof map for market-data analytics, dashboarding, validation discipline, and quant-adjacent research.
