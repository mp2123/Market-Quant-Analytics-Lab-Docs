# Privacy Boundaries

The value of this public docs hub depends on clear boundaries.

## Never Publish

- `.env` files;
- API keys, OAuth tokens, webhook secrets, broker credentials, or private keys;
- account-specific records;
- live bank or brokerage transaction data;
- SQLite runtime databases;
- local machine paths;
- private watchlists;
- Discord, n8n, TradingView, Streamlit, or monitor payloads;
- raw private research artifacts.

## Allowed After Review

- sanitized markdown;
- synthetic examples;
- public-source examples;
- screenshots using mock data;
- architecture diagrams;
- validation summaries;
- explanations of what remains private.

## Visitor Expectation

Hiring teams should see a clear technical story, not the private runtime. The public proof is the architecture, validation discipline, privacy judgment, and project narrative.
