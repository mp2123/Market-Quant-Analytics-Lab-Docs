# Source Governance

This ecosystem separates public proof from private implementation.

## Public-Safe Sources

- architecture summaries;
- sanitized examples;
- synthetic or public-data fixtures;
- diagrams that do not reveal secrets or local paths;
- validation notes and privacy boundaries;
- recruiter-facing project explanations.

## Private Sources

- live provider credentials;
- OAuth token stores;
- broker/API runtime state;
- SQLite databases;
- private watchlists;
- account-specific exports;
- local monitoring payloads;
- raw backtest outputs;
- production QuantStrat runtime details.

## Public Language Rule

Use market-data analytics, financial-data analytics, dashboarding, validation, and quant-adjacent research language.

Avoid public language that implies live broker automation, production trading authority, professional quant platform ownership, or account-specific investment advice.
