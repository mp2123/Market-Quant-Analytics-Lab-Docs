# Validation Notes

Before promoting any market / quant documentation, run a conservative review.

## Public Docs Review

- check repo visibility;
- scan for secret-shaped values;
- scan for local machine paths;
- scan for private provider names or account identifiers that should not be public;
- review language for overclaims;
- verify links point to public docs shells or portfolio pages, not private implementation repos;
- confirm screenshots use synthetic or reviewed public-safe data.

## Portfolio Review

- keep the primary portfolio title as `Institutional Market Data Engine`;
- keep `Python Market Data Dashboard` as a descriptive capability, not the top-level project name;
- explain QuantStrat ML as related research, not the same project;
- keep production QuantStrat runtime private/internal.

## Language Checks

Avoid:

- `production trading platform`;
- `high-frequency`;
- `live trading automation`;
- `institutional-grade trading system`;
- `professional quant trader`;
- `professional quant developer`;
- `professional quant researcher`;
- account-specific performance claims.
